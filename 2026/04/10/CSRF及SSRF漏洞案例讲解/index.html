CSRF及SSRF漏洞案例讲解

CSRF漏洞（跨站请求伪造）

盗用他人的身份去进行恶意攻击

靶场演示

进入pikachu 登录

再写一个攻击者的代码

此时 用户是处于登录状态 那么 在这个时候 用户在去访问攻击者给出的网站

这个网站会悄悄把数据包发送给攻击者写的代码地址

然后 用的信息会根据攻击者的数据进行修改

csrf漏洞检测

在修改信息的地方 然后抓包

然后 右键找到这个

然后会出现这样的弹窗 将下面的内容复制下来

将复制的内容 粘贴到个人的服务器上

如果 在去访问个人服务器

在这个页面 可以去查看网络是否有数据包

然后再去访问之前的网站刷新 如果信息在测试完之后 没有修改 那么就是没有csrf漏洞

防御

1.当用户发送重要的请求时需要输入原始密码

2.设置随机token

token值唯一 很难实现攻击

3.检验referer来源 不是很靠谱 也可以修改相同来源实现绕过

SSRF（服务器请求伪造）

目标 从外网无法访问内部系统

原因 由于服务端提供了从其他服务器应用获取数据的功能且没有对目标地址做过滤与限制

挖掘:

从web功能上寻找：

分享：通过url地址分享网页内容

转码服务：通过url地址把源地址的网页内容调优使其适合手机屏幕浏览

在线翻译：通过url地址翻译对应文本的内容。提供功能的国内公司有百度，有道等

图片下载与加载：通过url地址加载或下载图片

图片 文章收藏功能

未公开的api实现以及其他调用url 的功能

从url关键字中寻找：

share wap url link src source target u 3g display sourceURL imageURl domain

某些网站可能会出现远程上传图片 那么这时候可能是会出现ssrf

各个协议调用探针 http file dict ftp gopher 等

http://：探测内网主机存活、端口开放情况

gopher://：发送GET或POST请求；攻击内网应用，如FastCGI、Redis

dict://：泄露安装软件版本信息，查看端口，操作内网redis访问等

file://：读取本地文件
